# v2 명세서 — 다관점 스코어링 플랫폼 + 영속 데이터 레이어

> v1(MVP)은 RSI<30 → 단도 게이트 → 신호 알림의 단일 관점 일일 파이프라인이다. v2는 그 위에 **전수 영속 데이터 레이어**와 **5개 투자 프레임워크 스코어링**을 얹어, 같은 데이터로 여러 관점의 리서치·밸류에이션·랭킹을 산출한다.
> 작성 기준: 2026-06. v1 구현명세서와 함께 사용한다(중복 항목은 v1 참조).
> **v1 참조 소스 (GitHub):** v1 전체 코드 골격·구현은 저장소 https://github.com/Be-idler/Dhandho_RSI30 를, v1 구현명세서 문서는 같은 저장소의 `docs/`(https://github.com/Be-idler/Dhandho_RSI30/tree/main/docs)를 참조한다. 본 명세서에서 "v1 §X 참조"·"v1 A~G"·"v1 산식"은 모두 이 저장소의 v1 구현명세서를 가리키며, v2 코드가 v1 검증 로직(RSI·게이트·DART 파싱 등)을 이식할 때의 원본도 이 저장소다.
> 주의: 각 프레임워크 루브릭은 원저자 저작을 시스템화한 **해석(operationalization)**이며, 백테스트로 보정 대상이다. 캐논 출처는 각 저서(파브라이 『단도투자』, 손다이크 『아웃사이더』, 그린블랫 『주식시장의 보물찾기』, 베일리 기포드 LTGG 자료, 버핏/멍거 주주서한·강연)다.

---

## §1. 목적·범위

- v1 유지: 일일 RSI<30 → 단도 게이트 → 텔레그램 신호.
- v2 추가: ① 전수 영속 데이터 레이어, ② 5개 프레임워크(단도·LTGG·아웃사이더·마법공식·버핏멍거) 스코어링, ③ 자체 밸류에이션 엔진, ④ 관점별 랭킹·리포트.
- 핵심 원칙은 v1과 동일: 공식 출처 강제, 사실/유추/추정/불명 구분, 다출처 교차검증, 정반합 결론, 다중 밸류에이션, 최종 판단은 사람.

## §2. 데이터 레이어

### 트랙 구성 (2-트랙·2-봇)
- **트랙1 (메인·일일·봇1) — 단도 단일 관점**: 다트 적재(분기 prerequisite) → ② RSI<30 1차 → ③ 적재 재무로 단도 정량 게이트 2차 → ④ 통과 종목 정성 수집 + LLM → 신호. *RSI 진입 타이밍에 민감한 단도만 일일 루프에 둔다.*
- **트랙2 (별도·격주 주말·봇2) — 다관점 랭킹**: 나머지 4개 프레임워크(LTGG·아웃사이더·마법공식·버핏멍거) 전수 랭킹 → 관점별 상위 N 다이제스트. **RSI 게이트 없음**(랭킹 기반). 별도 텔레그램 봇으로 일일 신호와 분리.

### 계층
| 레이어 | 역할 | 주기 |
|---|---|---|
| **L-Q** | 정기공시 마감 다음 주 전 종목 정기보고서 다운로드·정규화(SSOT) | 연 4회 |
| **L-Δ** | RSI<30 등 트리거 종목의 최신 정정·수시공시 온디맨드 보강 | 매 영업일 |
| **L-Daily (트랙1)** | RSI 계산 → 단도 게이트 → 정성 → 신호(봇1) | 매 영업일 |
| **L-Multi (트랙2)** | 전수 DB 위 4개 프레임워크 랭킹 → 상위 N(봇2) | 격주 주말 |

### 수집 캘린더 (12월 결산 기준, "마감 다음 주")
- 사업보고서(연도+90일, ~3/31) → 4월 첫 주
- 1분기(분기+45일, ~5/15) → 5월 셋째 주
- 반기(반기+45일, ~8/14) → 8월 셋째 주
- 3분기(+45일, ~11/14) → 11월 셋째 주
- 수시공시(자사주·내부자·공급계약 등)는 일일/주간 별도 스캔.

### 저장·인프라
- **저장 백엔드: 네이버 클라우드(NCP)**. 가공 재무·공통 파생지표는 **Object Storage(S3 호환)에 Parquet**로 저장. **원시·XBRL 영구보관 불필요**(DART 재다운로드 가능). 상시 저장량은 약 1~3 GB 수준.
- **조회**는 DuckDB로 Parquet 직접 쿼리(DB 서버 0원). 규모·편의 확장 시 별도 분석 DB(예: BigQuery) 검토.
- **컴퓨트는 NCP 동일 리전에 co-locate**(Server VM 또는 Cloud Functions)해 egress·지연 최소화. 분기 전수 파싱은 무겁고 DART 호출이 잦음.
- DART OpenAPI 호출 한도 → 전수 배치는 재시도·체크포인트(중단 지점 재개).
- 저장 데이터는 개인 분석·보관 용도로 한정(공개 재배포는 약관·저작권 주의).

### 일일 파이프라인 타임라인 (확정)

**원칙**: 무거운 정량 필터는 저장된 가공 데이터로 끝내고, 비싼 작업(DART 원문 + LLM)은 통과한 극소수에만 건다. LLM은 **비동기 Batch API(50% 할인)** 로 제출 — *할인은 시각이 아니라 비동기 배치에서 발생*한다. **작업 시작은 08:30 UTC(KST 17:30, 장 종료 후)**, 알림은 그로부터 **13시간 후인 21:30 UTC(KST 06:30, 개장 전)**. **KRX 휴장일은 평일이어도 스킵**(`krx.is_trading_day`). 스케줄러는 UTC 기준이며, 트리거 A·B는 같은 UTC 날짜를 공유한다(일자 어긋남 방지).

| 시각(UTC) | 시각(KST) | 단계 |
|---|---|---|
| — | 15:30 | 한국 장 마감 (휴장일이면 전 단계 스킵) |
| — | ~16:30 | EOD 가격 시세 제공 |
| **트리거 A** 08:30 | 17:30 | ① RSI<30 1차 필터(EOD 가격) → ② NCP에서 가공 재무 로드 → **2차 정량 필터**(게이트 A·D 등) → ③ 통과 종목만 **정성 자료 수집**(DART 정정·수시공시·임원약력 + IR·인터뷰) → NCP 델타 업데이트 → ④ LLM 정성 분석을 **Batch API로 제출(async)** |
| (비동기 처리) | | LLM 배치 처리(보통 수십 분~수시간, 최대 24h) |
| **트리거 B** 21:30 | 06:30(익일) | ⑤ 배치 결과 수신·최종 신호 종합 → ⑥ **텔레그램 알림(개장 전)** |

- **2-트리거 구조**: 제출(A)과 알림(B)을 분리해, 비동기 배치 결과를 개장 전 정해진 시각에 안정적으로 전달. 트리거 B는 UTC 21:30 이후 → 모든 알림은 KST 06:30 이후·개장 전 도착.
- **멱등·실패 통보**: 각 단계 체크포인트, 실패 시 텔레그램 통보. 배치 미완료 시 트리거 B에서 대기·재시도.

### 데이터 흐름도

```
[분기] 전종목 보고서 → 가공(공통 파생지표 풀) → NCP Object Storage (Parquet)
                                                    │
[매일·트리거A/UTC 08:30, KST 17:30] RSI<30 1차 → NCP 가공재무 로드 → 2차 정량필터
                                                    │ (통과 소수만)
                        DART API: 정정·수시공시 수집 → NCP 델타 업데이트
                                                    │
                              LLM 정성분석 Batch 제출 (async, −50%)
                                                    │
[매일·트리거B/UTC 21:30] 배치결과 수신 → 신호 종합 → 텔레그램 알림(개장 전)
```

## §3. 3계층 데이터 모델 (정합성 핵심)

관점마다 원천을 중복 적재하지 않는다. 대신 3계층으로 분리한다.

1. **정규화 원천 (SSOT)** — `financials_normalized`: XBRL 파싱 결과를 **한 벌만** 적재. 단일 진실원천.
2. **공통 파생지표 풀** — `metrics_derived`: ROIC·FCF·EV/EBIT 등을 **한 번 계산해 모든 관점이 공유**(§4).
3. **관점별 스코어링 규칙** — `scores_framework`: 각 관점은 같은 지표 풀에서 *무엇을 고르고 어떻게 가중·임계·랭킹하는가*만 다르다.

→ 같은 지표는 항상 한 값(정합성 보장), 새 관점은 규칙만 추가.

### 스키마(요지)
- `financials_normalized(ticker, 기준분기, 계정, 값, 연결/별도)`
- `metrics_derived(ticker, 기준분기, 지표명, 값)`
- `filings_archive(rcept_no, ticker, 보고서유형, 접수일, 원문경로)` — 정정 버전관리
- `disclosures_event(ticker, 유형, 일자, 내용)` — 자사주·내부자·계약
- `scores_framework(ticker, 기준일, framework, 하위점수json, 종합, 등급)`
- `export_mapping(...)` — v1 D-OV 통관 매핑 유지

## §5. 프레임워크별 스코어링

각 프레임워크는 1.0~5.0 하위점수 → 가중 종합 → §11 등급. 정성 항목은 공식 출처 근거 강제, 미충족 시 2.5 상한+플래그.

### §5.1 단도투자 (Dhandho) — *기존 v1 A~G 사용*
"안 잃는 것" 우선. 하방보호(A)·밸류트랩 배제(D)가 게이트. 상세는 v1 구현명세서 §3~§8 참조. v2에서는 전수 DB·업종 백분위·자체 밸류에이션 엔진(§7)과 연결만 강화한다.

### §5.2 베일리 기포드 LTGG (Long-Term Global Growth) — 성장 관점

장기(5~10년) 비대칭 성장. **단도와 정반대 렌즈**(저평가가 아니라 성장 지속성·재투자수익이 핵심). 의도적 다변화.

| 하위지표 | 평가 | 데이터·점수화 | 가중 |
|---|---|---|---|
| **L1. 성장 활주로** | 매출 5y CAGR + 지속 가능성(시장침투율·TAM 여력) | CAGR≥20%&여력 큼→5 / 10~20%→3~4 / <10%→1~2 | 25% |
| **L2. 증분 자본수익(ROIIC)** | 재투자가 가치를 창출하는가 | ROIIC가 WACC 크게 상회·지속→5 / 근접→3 / 미달→1 | 20% |
| **L3. 해자 확장성** | 네트워크효과·규모·전환비용이 *커지는가* (정성) | 확장형 해자 입증→5 / 정태적→3 / 약함→1 | 20% |
| **L4. 경영진·문화·장기지향** | 창업자 주도, 미션, 장기 인센티브 (정성) | 강한 장기지향 입증→5 / 보통→3 / 단기·취약→1 | 15% |
| **L5. 단위경제·수익성 궤적** | 마진·단위경제 개선 추세 | 개선 추세 명확→5 / 횡보→3 / 악화→1 | 10% |
| **L6. 비대칭 옵셔널리티** | 하방 제한 대비 멀티배거 가능성 (정성) | 큰 옵셔널리티+하방한정→5 / 보통→3 / 비대칭 약함→1 | 10% |

**게이트:** L1·L2 ≥ 3.0(성장 지속성·재투자수익이 본령). **밸류에이션 게이트 없음** — 대신 §7 역DCF로 "현 주가가 함축하는 성장률이 달성 가능한가"를 점검.
**주의:** LTGG는 RSI<30 과매도 진입과 약하게 연동(§8). 성장 훼손 없는 일시 급락만 기회.

### §5.3 손다이크 아웃사이더 — 자본배분 관점

CEO의 핵심 직무는 자본배분. 외형이 아니라 **주당가치 성장**으로 측정.

| 하위지표 | 평가 | 데이터·점수화 | 가중 |
|---|---|---|---|
| **O1. 주당가치 성장** | 주당 FCF·BPS CAGR (외형 아님) | 높고 일관→5 / 보통→3 / 정체·희석→1 | 25% |
| **O2. 재투자·인수 규율** | 재투자 ROIC, M&A 후 ROIC | 고ROIC 규율적→5 / 평균→3 / 가치파괴 M&A→1 | 20% |
| **O3. 자사주 환원 품질** | 저평가 시 대규모 매입 + **소각 비율(한국조정)** | 저평가 매입+높은 소각비율→5 / 매입만·미소각 축적→2 / 고가매입·없음→1 | 20% |
| **O4. FCF 중심성** | 보고이익보다 FCF/주 추세 | FCF/주 견조 성장→5 / 보통→3 / 약화→1 | 15% |
| **O5. 간결한 본사·분권화** | 판관비/매출, 조직 구조 (정성) | 린·분권 입증→5 / 보통→3 / 비대·집권→1 | 10% |
| **O6. 오너십·독립적 사고** | 내부자 지분, 역발상 타이밍 (정성) | 높은 오너십+역발상→5 / 보통→3 / 낮음·추종→1 | 10% |

**게이트:** O1·O3 ≥ 3.0.
**한국조정(중요):** 자사주를 **소각하지 않고 쌓아두면 감점**(터널링·경영권 방어 리스크). **소각 비율**이 높을수록 진짜 아웃사이더형 규율로 가점. 저배당은 규율의 증거가 아니라 터널링 신호일 수 있으므로 환원의 *질*로 판정.

### §5.4 그린블랫 마법공식 — 순수 퀀트

두 지표 랭킹 합산. 전수 DB가 있어야 성립(랭킹은 전 종목 비교).

- **이익수익률(EY)** = EBIT / EV(시총+순차입금) — 높을수록↑
- **자본수익률(ROC)** = EBIT / (순운전자본 + 순고정자산) — 높을수록↑
- 전 종목을 EY·ROC 각각 순위 → **순위 합산** → 상위 랭킹.
- **제외**: 금융·유틸리티, 최소 시총 미달 종목.
- **점수화**: 합산순위 백분위 → 1.0~5.0 (상위 10%→5 … 하위→1).
- **갱신**: 분기 배치. 트레일링 기준.

**주의:** 순수 기계식이라 밸류트랩·일회성 EBIT 왜곡에 취약 → 단도 D섹션·버핏 B섹션과 교차 확인 권장(합의 플래그, §6).

### §5.5 버핏·멍거 가치투자 — 품질+적정가 관점

"훌륭한 기업을 적정가에." 단도보다 **품질·해자·복리**를 더 중시(자산 바닥보다 지속 수익력).

| 하위지표 | 평가 | 데이터·점수화 | 가중 |
|---|---|---|---|
| **B1. 해자 내구성** | 정성 + GP/A·마진 안정성 | 입증된 내구 해자→5 / 보통→3 / 약함→1 | 20% |
| **B2. ROE/ROIC 일관성** | 10년 평균·변동성 | 10y ROE≥15% 꾸준→5 / 변동 큼→3 / 낮음·불안정→1 | 20% |
| **B3. 재무 보수성** | 낮은 레버리지 | 무차입·저부채→5 / 보통→3 / 과다→1 | 12% |
| **B4. 오너어닝/FCF 예측가능성** | FCF 안정·예측가능 | 높고 안정→5 / 보통→3 / 불안정→1 | 15% |
| **B5. 가격결정력** | 매출총이익률 안정·상승 | 견고·상승→5 / 횡보→3 / 잠식→1 | 13% |
| **B6. 경영진 정직성·자본배분** | 정성 + 한국조정(지배지분순익·소수주주 정렬) | 정직·정렬 입증→5 / 보통→3 / 터널링 정황→1(하드 배제) | 12% |
| **B7. 안전마진** | §7 내재가치 대비 | 큰 할인→5 / 적정→3 / 고평가→1 | 8% |

**게이트:** B1·B2·B6 ≥ 3.0. B6에 터널링·횡령·소수주주 침해 확인 시 **즉시 제외**(밸류에이션 무관).

## §6. 크로스-프레임워크 종합 (점수 평균 금지)

> **전달 경로**: 단도는 **트랙1(일일·봇1)** 신호로, 나머지 4개는 **트랙2(격주·봇2)** 랭킹 다이제스트로 분리 전달한다. 아래 종합(합의/상충)은 트랙2 다이제스트나 수동 분석에서 활용한다.

**5개 점수를 하나로 평균하지 않는다.** 각 프레임워크는 서로 다른 것을 측정하며, 의도적으로 불일치한다(단도 심층가치 ↔ LTGG 성장). 평균은 무의미한 중간값을 만든다.

대신:
1. **관점별 독립 판정·랭킹**: 각 종목은 5개 프레임워크 각각에서 등급·순위를 받는다. 산출물은 `종목 × 5개 관점` 매트릭스 + 관점 태그.
2. **합의 플래그(고확신)**: 둘 이상 관점에서 "분할 매수" 이상이면 고확신 후보로 태깅(예: 단도+버핏 동시 통과 = 견고한 가치, 마법공식+버핏 = 퀀트·정성 합치).
3. **상충 표면화**: 한 관점은 매수, 다른 관점은 제외인 경우 그 *이유*를 함께 제시(예: LTGG 매력 but 단도 하방보호 미달) → 사람이 판단.

## §7. 자체 밸류에이션 엔진 (보류)

> **현재 보류.** 아래 설계는 향후 재개 시 사용한다. **보류 기간 중 밸류에이션은** 각 프레임워크에 내장된 멀티플(EV/EBIT·PER 자기밴드·업종 대비)과 자산 플로어(순현금·NCAV)로 처리하며, 정밀 DCF/EPV/역DCF 산출은 수행하지 않는다. (버핏 B7 안전마진도 보류 기간엔 멀티플·자산 기준으로 판정.)

전수 단계는 가정이 적은 견고한 방법, finalists만 정밀.

- **전수(견고)**: ① 멀티플(EV/EBIT·PER 자기밴드·업종 대비) ② **자산 플로어**(순현금·NCAV → 하방 앵커) ③ **EPV**(무성장 정상화이익/WACC, 성장 예측 불필요) ④ **역DCF**(현 주가가 함축하는 성장률 역산 → 보수성 판정).
- **finalists(정밀)**: 게이트 통과 소수에만 **시나리오 DCF**(bear/base/bull + 확률가중) → 비대칭비율 산출.
- **산출물**: 목표가 = **레인지**(보수 base ~ bull). 진입가 = **보수 내재가치 × (1 − 안전마진)**, 그리고 **자산 플로어로 하방 앵커**.
- **증권사 목표가**: 입력이 아니라 교차검증·소수의견 참고용.
- **필수**: WACC·영구성장률 가정 명시 + 민감도 표 동반(정밀한 오답 방지).
- **역DCF를 1차 권장**: 예측 부담 없이 단도/버핏의 "시장이 함축한 기대 vs 보수적 현실"을 직접 점검.

## §9. 한국 특수조정 (전 관점 공통)

- **자사주는 소각 여부가 핵심**: 미소각 축적 = 터널링·방어 리스크(감점), 소각비율 높음 = 진짜 환원(가점). 3차 상법 개정(1년 내 의무소각, v1 §7) 촉매와 연결.
- **저배당 ≠ 규율**: 한국에선 저배당이 터널링·밸류트랩 신호일 수 있음 → 환원의 *질*로 판정.
- **지주·NCI 구조**: 연결 순이익이 아니라 **지배지분 순이익**으로 평가. NCI 과다 구조는 모회사 실질 수익을 가릴 수 있음.
- **자산 ≠ 접근가능 가치**: 자산가치가 소수주주에게 환원되지 않는 사례(한국공항형) 경계.
- **밸류업 공시**: 루머가 아니라 확정 공시를 촉매로 우선.
- **상법 촉매**: v1 §7 타임라인 테이블 적용(자사주 의무소각·집중투표·이사 충실의무 등).

## §10. LLM 사용 (정성 항목)

- 정성 하위지표(LTGG L3·L4·L6, 아웃사이더 O5·O6, 버핏 B1·B6, 단도 B4·D2·E2·F3 등)에 적용.
- **Haiku로 원문에서 관련 구절 추출(대량·저렴) → Sonnet으로 판단 스코어링(균형)**.
- **게이트 통과 finalists로 한정**, **비동기 Batch API(50% 할인) + 루브릭 프롬프트 캐싱(최대 90%)**, 출력은 압축 JSON + max_tokens 제한.

### 정성 자료 소스 (확장)
3단계에서 finalists에 한해 아래 자료를 수집해 분석 질을 높인다.

| 자료 | 수집처 | 매핑 | 소스 계층 |
|---|---|---|---|
| **경영진 약력·이력** | DART 사업보고서 '임원 현황'(약력·재직·주식소유), 등기 | F2·F3, 아웃사이더 O6, 버핏 B6 | 1차(DART) |
| **IR 자료** | 기업 IR(실적발표·기업설명회), KIND, 한국IR협의회, 기업 홈페이지 | E(촉매·밸류업), LTGG L4, 버핏 B1 | 2차(IR) |
| **경영진 인터뷰·언론** | 캐피탈마켓 미디어(더벨·딜사이트 등), 주요 언론 | LTGG L4(장기지향), 단도 E·F 보조 | 3차(미디어) |

### 정성 분석 규율 (필수)
- **소스 계층 태깅**: 모든 정성 주장에 출처 계층(1차 DART > 2차 IR > 3차 미디어) 표시. **DART·IR 우선**, 미디어는 보조.
- **인터뷰·기사는 '사실'이 아니라 '경영진의 주장·방향성'으로 취급**(자기보고 편향 인지). 검증 가능한 행동(공시·실적)으로 교차확인.
- **근거 강제**: 모든 정성 점수에 출처(문서·URL·일자) 첨부. 근거 없으면 2.5 상한+플래그. 블로그·포럼·검증불가 소셜은 제외.
- **저작권**: 기사·IR 원문을 복제·재배포하지 않고 사실·요지만 추출(개인 분석 용도).
- **비용**: 자료 추가로 입력 토큰↑ → finalists 한정·Haiku 추출·캐싱으로 관리.
- **최종 판단은 사람.**

## §11. 산출물·등급

- 등급(관점별, 5단계): **적극 매수 → 분할 매수 → 관심종목 편입 → 보류 → 제외**.
- 종합 임계(제안): 종합 ≥ 4.0 & 게이트 통과 → 적극/분할, 3.0~4.0 → 관심, 게이트 미통과 → 보류, < 3.0 또는 하드배제 → 제외. (프레임워크별 백테스트로 보정.)
- 리포트 형식(v1 보고 시퀀스 준용): 한 줄 verdict → 출처 → 요약 → 체크리스트 → 사업분석 → 재무안정성 → 밸류에이션 → 안전마진 → 촉매 → 리스크 → 시나리오 → 정반합 결론 → 최종 판단 → 불명확 목록.

## §12. 리스크·한계

- **프레임워크 충돌은 버그가 아니라 기능**: 평균화 금지, 상충은 사유와 함께 표면화.
- **마법공식 기계식 취약**: 일회성 EBIT·밸류트랩 → 단도 D·버핏 B와 교차.
- **정성 환각**: 근거 강제·사람 검증으로 통제.
- **미디어·인터뷰 편향**: 인터뷰·기사는 자기보고 편향·환각 위험 → 소스 계층 태깅·행동(공시·실적) 교차검증·사람 검증으로 통제.
- **DCF 민감도**: 자체 밸류에이션 엔진(§7)은 현재 보류. 재개 시 가정 명시·민감도 표 필수, 전수 자동 DCF 지양(역DCF·EPV 우선).
- **데이터 신선도**: 분기 사이엔 수시공시·정정만 델타 반영.
- **백테스트 필수**: 모든 가중·임계는 과거 검증 후 보정. 미검증 임계로 실매매 판단 금지.

## §13. 부록 — 단도 A·C·D 구현 정의 (코드 매핑)

> 클로드 코드 구현용. 필드명은 `dart.normalize_financials` / `metrics.compute_derived` 출력 키.
> 임계는 v1 구현명세서 기준 제안값(백테스트로 보정).

### 13.0 공통 규칙
- **근거불충분**: 입력이 `None`이면 해당 하위지표 **2.5 상한 + 플래그**. 게이트 섹션(A·D)에 플래그가 끼면 신호는 **WATCH 강등**.
- **업종 상대평가**: 금융·지주·리츠는 절대 임계 대신 `sector_relative.percentile_score(value, peers, higher_is_better)`로 대체. 비교군 5종목 미만이면 전체시장 폴백 또는 근거불충분.
- **2단계 게이트**:
  - **정량 사전필터(트리거 A, LLM 이전)**: 계산 가능한 정량 하위지표만으로 `A_quant`·`D_quant`를 산출해 finalists를 좁힌다. 정성(D2)·다년 미확보 항목은 이 단계에서 제외.
  - **최종 게이트(트리거 B, LLM 이후)**: D2 등 정성 포함 전체 A·D로 `A ≥ 3.0 AND D ≥ 3.0` 적용.

### 13.1 A 섹션 — 하방보호 (25%)
| 하위(가중) | 입력 필드 | 산식 | 1.0~5.0 매핑 | 단계 |
|---|---|---|---|---|
| A1 순현금비율(35%) | `net_cash_to_mktcap` | net_cash/시총 | ≥0.40→5 / 0.25~0.40→4 / 0.10~0.25→3 / 0~0.10→2 / <0→1 | 정량 |
| A2 NCAV/시총(25%) | `ncav_to_mktcap` | (유동자산−총부채)/시총 | ≥1.0→5 / 0.67~1.0→4 / 0.33~0.67→3 / 0~0.33→2 / <0→1 | 정량 |
| A3 재무건전성(25%) | `interest_coverage`, `debt_ratio` | 이자보상배율 기본 + 부채비율 보정 | 무차입·≥5→5 / 3~5→4 / 1.5~3→3 / 1~1.5→2 / <1→1. 부채비율>2.0 −1.0, <0.5 +0.5 (1~5 클리핑) | 정량 |
| A4 FCF안정성(15%) | `fcf`(최근 5년) | FCF<0 연도수 n | n=0→5 / 1→4 / 2→3 / 3→2 / ≥4→1 | 정량(다년) |

`A = 0.35·A1 + 0.25·A2 + 0.25·A3 + 0.15·A4`. **게이트 A ≥ 3.0.**
업종 왜곡 시: A1→`net_cash_to_mktcap` 업종 백분위, A2→지주는 NAV할인·리츠는 LTV/배당 백분위로 치환.

### 13.2 C 섹션 — 밸류에이션 (20%)
| 하위(가중) | 입력 필드 | 산식 | 매핑 | 단계 |
|---|---|---|---|---|
| C1 트레일링 멀티플(35%) | `ev_ebit`, `per` (적자 시 `pbr`,`psr`) | EV/EBIT·PER 각 업종 백분위 점수의 평균(저평가일수록↑) | 백분위 매핑(§2). 둘 다 무효(적자)면 PBR·PSR 폴백 | 정량 |
| C2 자기밴드 위치(25%) | `ev_ebit` 또는 `pbr` 5년 시계열 | 현재값의 자기 5년 밴드 백분위 | 하위20%ile↓→5 … 상위→1 | 정량(다년) |
| C3 이익추세 proxy(25%) | `operating_income` TTM | YoY·QoQ 증감 | YoY+20%↑&QoQ+→5 / 증익→4 / 횡보→3 / 감익→2 / 적자전환→1 | 정량(다기) |
| C4 과도낙폭(15%) | 52주 고점 대비 하락률 | D 구조판정과 교차된 '과매도분'만 | −25~−45% 구간 고점, D1·D2와 교차 | 정량+교차 |

`C = 0.35·C1 + 0.25·C2 + 0.25·C3 + 0.15·C4`.
**주의**: `operating_income`/EBIT 음수면 EV/EBIT·PER 무효 → PBR·PSR 폴백 또는 근거불충분.

### 13.3 D 섹션 — 밸류트랩 배제 (15%, 게이트)
| 하위(가중) | 입력 | 산식·판정 | 매핑 | 단계 |
|---|---|---|---|---|
| D1 구조적 추세(30%) | `revenue`,`operating_income` 5년 | 매출 CAGR + 영업이익 회귀 기울기 | 둘 다 성장/유지→5 / 한쪽 둔화→3~4 / 둘 다 구조적 하락→1~2 | 정량(다년) |
| D2 급락 원인(30%) | 공시·뉴스(LLM) | 일회성 vs 구조적 | 일회성→5 / 혼재→3 / 영구훼손→1 | **정성(LLM)** |
| D3 산업 사양화(20%) | 업종·전방수요(LLM/매핑) | 수명주기 | 성장/성숙안정→4~5 / 성숙후기→3 / 사양→1~2 | 정성/반정량 |
| D4 재무 생존력(20%) | 감사의견·자본잠식(DART) | 계속기업·자본잠식 | 적정+건전→5 / 강조·부분잠식→2 / 계속기업불확실·자본잠식→1 | 정량(공시) |

`D = 0.30·D1 + 0.30·D2 + 0.20·D3 + 0.20·D4`. **게이트 D ≥ 3.0.** D-OV 통관 오버레이(v1 §6)는 자격 종목만 D1·D2 ±0.5 보정.
**정량 사전필터의 D_quant**: D2(정성)를 뺀 D1·D4 중심으로 보수적 산출(미확보는 2.5캡). 최종 D는 트리거 B에서 D2 포함 완성.

### 13.4 2단계 게이트 의사코드

A_quant = wavg(A1,A2,A3,A4)                 # 전부 정량

sections = {A, B, C, D(완성), E, F}
